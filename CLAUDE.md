## Project Overview

Tree-sitter grammar for the F\* proof-oriented programming language. Generates a parser from `grammar.js` with an external C scanner for nested block comments.

## Commands

```sh
tree-sitter generate   # Generate parser from grammar.js (produces src/parser.c, src/grammar.json, src/node-types.json)
tree-sitter test       # Run all test corpus files in test/corpus/
tree-sitter parse <file>  # Parse a specific F* file (e.g., tree-sitter parse examples/Factorial.fst)
```

Nix users: `nix develop` provides tree-sitter CLI, Node.js, Neovim with the parser pre-configured, and pre-commit hooks (nil, statix, treefmt with nixfmt/prettier/clang-format).

## Architecture

- **`grammar.js`** — Main grammar definition. Defines all F\* syntax rules with multiple operator precedence levels. This is the primary file to edit when changing the parser.
- **`src/scanner.c`** — External scanner handling nested block comments `(* ... (* ... *) ... *)`. Referenced via `externals` in grammar.js.
- **`src/parser.c`, `src/grammar.json`, `src/node-types.json`** — Auto-generated by `tree-sitter generate`. Do not edit manually.
- **`test/corpus/`** — Test cases in tree-sitter corpus format (input followed by expected S-expression parse tree). Split by category: comments, declarations, expressions, patterns, types.
- **`queries/highlights.scm`** — Neovim-compatible syntax highlighting queries.

## Grammar Structure

Key rule categories in `grammar.js`:

- **Module system**: `module`, `open`, `include`, `friend`, module abbreviations
- **Declarations**: `val`, `let`/`let rec`, `type` (inductive, record, abbreviation), `exception`, `assume`, `effect`, `class`
- **Expressions**: control flow, operators, application, literals (int with hex/octal/binary, strings, chars), constructors, calc blocks, refinement types
- **Patterns**: constructor, cons, or, as, typed, tuple, list, record, wildcard
- **Binders**: explicit `(x:t)`, implicit `#x` / `{x:t}`, typeclass `{|x:t|}`
- **Identifiers**: `lid` (lowercase), `uid` (uppercase), `uid_path` (qualified), `uid_path_dot_lid` (qualified with lowercase tail)

The grammar has 9 declared conflicts for ambiguous parses between patterns/terms, identifiers, and field assignments.

## Workflow

After editing `grammar.js` or `src/scanner.c`, always run `tree-sitter generate` then `tree-sitter test` to verify changes. Add test cases in `test/corpus/` for any new or modified grammar rules.

Follow [Angular Conventional Commits](https://www.conventionalcommits.org/). Example types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
